# Dev Flow Lite (≈ 40 lines)

## Overview
This file provides a concise, step‑by‑step development flow that the Cline agent follows when executing a task. It replaces the heavyweight `docs/onitask_Dev_Flow_7_2.md` for routine operations while still respecting the full process defined in the master spec.

## Steps
1. **Initialize Context**
   - Read `.clinerules/00-memory-bank.md` to load the Memory Bank mapping.
   - Load `docs/onitask_product_vision.md`, `docs/ARCHITECTURE-COMPACT.md`, `docs/onitask_dev_setup.md` and `docs/TASKS.md` (which now also holds `activeContext`).

2. **Identify First Unblocked Task**
   - Scan `docs/TASKS.md` for the first `- [ ]` entry without a `@blocked_by` tag.
   - Write its identifier and description to `activeContext.md` (or the `activeContext` section in `docs/TASKS.md`).

3. **Gather MOC (Map of Context)**
   - Resolve any `MOC-<tag>` references from the task description.
   - Pull relevant architecture snippets from `ARCHITECTURE-COMPACT.md`.

4. **Plan** (Plan Mode)
   - Generate a short plan (max 5 steps) describing the actions needed.
   - Present the plan to the user for approval.

5. **Execute** (Act Mode)
   - Run the actions sequentially (e.g., code generation, tests, migrations).
   - After each action, update the task status in `docs/TASKS.md`.

6. **Testing & Verification**
   - Run `npm test` and `tsc --noEmit`.
   - If any test fails, increment `retry_count` in `activeContext` and repeat from step 4.

7. **Commit & Review**
   - Commit changes with a message referencing the task ID.
   - Open a diff for the Shadow Evaluator (Antagonist) to review.

8. **Finalize**
   - Mark the task as completed (`- [x]`) in `docs/TASKS.md`.
   - Clear `activeContext` (reset `retry_count`).

## Notes
- The `pg-aiguide` MCP server defined in `cline_mcp_settings.json` is used for any database‑related queries.
- All environment variables (e.g., `DATABASE_URL`) are read from the MCP server configuration.
- This lightweight flow is intended for routine tasks; for complex architectural changes, refer back to the full `docs/onitask_Dev_Flow_7_2.md`.
